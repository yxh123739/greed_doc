# Transit Download Bundle — Design Spec

## Overview

Add a "Download" feature to the transit analysis page (`/benchmark/transit`) that generates a ZIP bundle containing three deliverables for LEED v5 BD+C LTc3 submission:

1. **transit-map.png** — Static map screenshot with project location, qualifying station markers, and walking route polylines
2. **trip-summary.pdf** — Dynamically generated weekday/weekend trip count table matching LEED Quality Transit format
3. **mta-schedule-*.pdf** — Official MTA printable subway schedule PDFs for all qualifying route groups

## Architecture

**Approach: Fully Server-Side (single API endpoint)**

A single `POST /api/transit/download` endpoint handles all generation, fetching, and packaging. The frontend sends the transit analysis results; the server returns a streamed ZIP file.

### Request / Response Flow

```
Browser ──POST──▶ /api/transit/download
  Body: { address, qualifyingStations, transitScore, totalWeekdayTrips, ... }

  Server:
    ├─ 1. Google Static Maps API → transit-map.png
    ├─ 2. Supabase Storage fetch → mta-schedule-*.pdf
    └─ 3. PDFKit generate → trip-summary.pdf
    (Steps 1-3 run in parallel via Promise.all)

    4. Archiver packs all files → ZIP stream

/api/transit/download ──ZIP stream──▶ Browser
  Content-Disposition: attachment; filename="LEED-Transit-Report.zip"
```

### ZIP Bundle Structure

```
LEED-Transit-Report/
  transit-map.png        — 1280x800 @2x static map
  trip-summary.pdf       — Generated trip count table
  mta-schedule-1-2-3.pdf — Matched MTA schedule(s)
  mta-schedule-N-Q-R-W.pdf
```

## Component Details

### 1. Static Map Screenshot (`transit-map.png`)

**API:** Google Maps Static API

- Size: `640x400`, scale `2` (actual 1280x800 retina)
- Project location: blue pin marker
- Qualifying stations: purple pin markers with numeric labels (1, 2, 3...)
- Walking routes: encoded polylines from Google Directions API for each qualifying station
- 0.5 mi reference circle: pre-computed encoded polyline (64-point arc approximation)

**Implementation location:** `lib/static-map.ts`

### 2. MTA Schedule PDFs (`mta-schedule-*.pdf`)

**Storage:** Supabase Storage bucket `mta-schedules`

Pre-downloaded MTA Printable Subway Schedules, organized by route group:

```
mta-schedules/
  1-2-3.pdf
  4-5-6.pdf
  7.pdf
  A-C-E.pdf
  B-D-F-M.pdf
  G.pdf
  J-Z.pdf
  L.pdf
  N-Q-R-W.pdf
  S.pdf
  SIR.pdf
```

**Route-to-PDF mapping (constant):**

```typescript
const ROUTE_TO_SCHEDULE: Record<string, string> = {
  "1": "1-2-3", "2": "1-2-3", "3": "1-2-3",
  "4": "4-5-6", "5": "4-5-6", "6": "4-5-6",
  "7": "7",
  "A": "A-C-E", "C": "A-C-E", "E": "A-C-E",
  "B": "B-D-F-M", "D": "B-D-F-M", "F": "B-D-F-M", "M": "B-D-F-M",
  "G": "G",
  "J": "J-Z", "Z": "J-Z",
  "L": "L",
  "N": "N-Q-R-W", "Q": "N-Q-R-W", "R": "N-Q-R-W", "W": "N-Q-R-W",
  "S": "S",
  "SIR": "SIR",
};
```

**Matching logic:** Extract all `routeName` values from `qualifyingStations[].routes[]` where `counted === true` → deduplicate via mapping → fetch unique PDF files from Supabase Storage.

**Implementation location:** `lib/mta-schedules.ts`

### 3. Trip Summary PDF (`trip-summary.pdf`)

**Library:** `pdfkit` (server-side Node.js PDF generation)

**Layout (Letter size, portrait):**

- **Header:** "LEED v5 BD+C — LTc3 Access to Transit" title, "Quality Transit Calculation — Option 2: Subway" subtitle, green (#1a5632) accent
- **Project Info block:** Address, analysis date, walking radius (0.5 mi for subway)
- **Qualifying Transit Routes table:**
  - Columns: #, Station, Route, Type, Walking Distance, Weekday Trips, Weekend Trips
  - Route column displays MTA line name (e.g., "N", "Q", "R")
  - Alternating row backgrounds for readability
  - Footer row: TOTAL with summed weekday/weekend trips
- **LEED Score box:** Large score display (e.g., "4 / 4 Points"), threshold met, credit reference
- **Footer:** "Generated by Anchor Sustainability LEED Feasibility Tool" + "Source: MTA GTFS Static Feed"

**Implementation location:** `lib/trip-summary-pdf.ts`

### 4. ZIP Packaging

**Library:** `archiver` (streaming ZIP creation)

All three components are generated in parallel (`Promise.all`), then added sequentially to the archive. The archive is piped directly to the HTTP response as a stream (no temp files).

**Implementation location:** `app/api/transit/download/route.ts`

### 5. Frontend UX

**Location:** `CtaPanel` component in `app/benchmark/transit/page.tsx`

- Download button click → POST to `/api/transit/download` with transit data payload
- During generation: button shows loading spinner + "Generating report..."
- On success: trigger browser download via `URL.createObjectURL` + programmatic `<a>` click
- On failure: `toast` with server-returned error message

**Payload shape (sent from frontend):**

```typescript
interface DownloadRequest {
  address: string;
  city: string;
  stateProvince: string;
  zipCode: string;
  country: string;
  geocodedLocation: { lat: number; lng: number };
  qualifyingStations: TransitStation[];
  totalWeekdayTrips: number;
  totalWeekendTrips: number;
  transitScore: number;
}
```

## Error Handling

**Strategy: Fail-fast with specific error messages.**

Any step failure immediately aborts the entire operation and returns a descriptive error to the user.

| Failure Point | HTTP Status | Error Message Example |
|---|---|---|
| Static Maps API error | 502 | "Map generation failed: Static Maps API returned 403" |
| Directions API error (polylines) | 502 | "Walking route generation failed for station '57 St - 7 Av'" |
| MTA PDF not in Storage | 404 | "Schedule not found: mta-schedule-A-C-E.pdf is missing from storage" |
| Supabase Storage unreachable | 502 | "Failed to fetch MTA schedule: Supabase Storage connection timeout" |
| PDFKit generation error | 500 | "Trip summary PDF generation failed" |
| Archiver error | 500 | "ZIP packaging failed" |

Frontend displays the error via `toast(errorMessage)`.

## Dependencies (new)

| Package | Purpose | Size |
|---|---|---|
| `pdfkit` | Server-side PDF generation | ~1.5 MB |
| `archiver` | Streaming ZIP creation | ~200 KB |

No new client-side dependencies.
